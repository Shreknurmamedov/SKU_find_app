Не отправляйте пароли и токены прямо в чат. Лучше использовать `.env`, менеджер секретов или отдельные приглашения в сервисы.

## Сейчас

- GitHub/GitLab репозиторий, если код нужно вести удаленно.
- Список собственных брендов.
- Каталог SKU в CSV/XLSX/JSON.
- Примеры фото и видео торговых точек.

## Как дать доступ к GitHub

Самый простой вариант:

1. Создайте репозиторий на GitHub.
2. Откройте `Settings` -> `Collaborators and teams`.
3. Нажмите `Add people`.
4. Введите GitHub username исполнителя или email, который он даст отдельно.
5. Выдайте роль `Write` для обычной разработки или `Admin`, если нужно настраивать Actions, secrets и protected branches.

Если репозиторий в организации, доступ обычно выдается через команду: `Organization settings` -> `Teams` -> нужная команда -> `Add member`.

Не отправляйте GitHub password или personal access token в чат. Если нужен deploy/token, лучше добавить его в `Settings` -> `Secrets and variables` -> `Actions`.
