# Разметка датасета

Для этой задачи нужна не только разметка бренда/SKU. Особенно важна связь одного физического товара между кадрами.

## Обязательные поля

- `image_id` или `frame_id`
- `object_instance_id`
- `same_physical_item_id`
- `bbox`
- `mask_path` при наличии сегментации
- `brand_id`
- `sku_id` или `UNKNOWN`
- `is_own_brand`
- `visible_area_ratio`
- `occlusion_level`
- `recognition_status`
- `evidence_quality`

## Статусы

- `recognized`: SKU подтвержден.
- `brand_only`: бренд подтвержден, SKU нет.
- `unknown`: товар виден, но бренд/SKU не подтверждены.
- `needs_retake`: зона требует пересъемки.
- `not_product`: область не является товаром.

## Почему нужен `same_physical_item_id`

Без этого поля нельзя измерить двойной подсчет. Если один и тот же товар виден на 20 кадрах, все 20 наблюдений должны ссылаться на один физический объект.

## Минимальная партия для старта

- 30-50 видео реальных торговых точек.
- 5-10 брендов.
- 100-300 SKU.
- Разные условия: блики, перекрытия, старые упаковки, хаотичная выкладка.
