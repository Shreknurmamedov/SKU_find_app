# Ночной прогон самоулучшения

Цель: вечером заполнить разметку, запустить один скрипт, утром прочитать отчёт
«было/стало» и применить лучший конфиг.

## Что он делает (3 стадии, каждая изолирована)

1. **Перебор детектора.** Прогоняет полный SKU-аудит каждого размеченного видео
   под несколькими конфигами детектора (веса × `conf`), считает SKU-level
   precision/recall/F1 против вашей разметки и выбирает лучший. Метрика
   агрегируется по видео через TP/FP/FN, поэтому SKU, найденный не в том ролике,
   остаётся ошибкой, а не ложным совпадением.
2. **Классификатор бренда.** Строит датасет из 14k эталонных фото (по бренду,
   8 классов) и обучает визуальный распознаватель бренда (`yolo classify`).
   Сообщает val top-1 точность. В прод НЕ подключается автоматически —
   решение по итогам утреннего отчёта.
3. **Отчёт.** Пишет `reports/overnight/SUMMARY.md` (таблица F1, лучший конфиг,
   точность классификатора) и `reports/overnight/best_config.json`.

## Шаги

### 1. Заполнить разметку (вечером, ~1 час) — это единственная ручная часть

`data/eval/sku_presence.csv`, по строке на видео:

```csv
video,expected_sku_ids
IMG_8886.MOV,HUTER_64_1_20_DY5000LX_DY6500LX|RESANTA_65_3_SAI_220
IMG_8882.MOV,
```

- `expected_sku_ids` — реально присутствующие на полке `sku_id` из
  `data/catalog/own_products.csv`, через `|`. Пусто = на видео нет наших SKU.
- Метрика — присутствие УНИКАЛЬНЫХ SKU, не количество копий.
- Чем больше видео (20–30), тем надёжнее измерение.

### 2. Запустить на ночь

```bash
./scripts/run_overnight.sh
# переживает закрытие терминала; лог: reports/overnight/run.console.log
```

Переменные: `SKU_AUDIT_DEVICE` (mps/cpu), `EPOCHS` (по умолчанию 40),
`TRUTH` (путь к CSV).

### 3. Утром

```bash
open reports/overnight/SUMMARY.md
```

Если лучший конфиг бьёт текущий прод (`v6_c40`) по F1 — применить его веса/`conf`
(см. `best_config.json`) в backend и/или экспортировать в APK.

## Честные ограничения

- **Без разметки скрипт не запустится** — измерять не против чего.
- Детектор переобучается только при наличии новых проверенных bbox (отдельный
  путь, см. `docs/detector-v8-workflow.md`); этот прогон НЕ переобучает детектор,
  а подбирает лучший из существующих весов/порогов.
- Классификатор бренда учится на студийных эталонных фото — на реальных полках
  (другой ракурс/свет) прирост не гарантирован; смотри val top-1 как ориентир.

## Быстрая проверка плумбинга (без долгого прогона)

```bash
python3 -m ml.overnight_improve --truth data/eval/sku_presence.csv --smoke
# 1 конфиг, reuse треков, 1 видео, 1 эпоха
```
