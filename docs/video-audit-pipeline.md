# Видео-аудит SKU: пайплайн

Двухэтапная схема подсчета SKU по видео полки. Детектор не знает брендов — он
только находит товар; бренд/модель/группа определяются по кропу из каталога.

```
видео ─► [1] детекция+трекинг ─► [2] re-ID ─► [3] распознавание ─► отчет
         (ByteTrack: 1 трек        (склейка       (OCR + каталог)     brand/
          = 1 объект)               повторных                        category/
                                    проходов)                         model + review
```

## Этап 1 — детектор товара (`ml/train_detector.py`, `ml/track_video.py`)

- Один класс `product`. Датасет собирается из старой 2-классовой разметки:
  `python3 ml/build_product_dataset.py` → `ml/datasets/sku_products`.
- Обучение: `python3 ml/train_detector.py` (yolo11n, MPS). Веса →
  `weights/product_det.pt`. Текущий baseline: mAP50 ≈ 0.44.
- Трекинг решает двойной счет **между кадрами**: ByteTrack дает стабильный id,
  один трек = один физический товар. Для каждого трека сохраняется лучший кроп.

## Этап 2 — re-ID (`ml/reid_merge.py`)

Склеивает треки, возникшие при **повторном проходе** камеры. Сливает только если:
треки не пересекаются во времени, между ними реальный временной разрыв
(`min_gap_frames`) и кропы визуально похожи (`sim_threshold`, по умолчанию 0.96).
Консервативно намеренно: на одиночном проходе слияний ≈ 0, рядом стоящие
одинаковые товары остаются разными объектами. Эмбеддинги — из backbone YOLO
(офлайн; `mobileclip_blt.ts` в репозитории битый, см. память проекта).

## Этап 3 — распознавание (`ml/sku_recognize.py`)

OCR кропа (easyocr ru+en) с перебором 4 поворотов (фото повернуты, OCR читает
только горизонтальный текст) → матчинг с `data/catalog/own_products.csv`:
сначала по коду модели, затем по бренду. Бренд/модель/группа берутся из строки
каталога. Уверенность: код модели ≈ 0.78–0.9, только бренд ≈ 0.6, нет совпадения
→ unknown.

## Запуск аудита (`ml/audit_video.py`)

```bash
python3 -m ml.audit_video --video "ТТ Пэкстрой/IMG_8886.MOV" \
    --weights weights/product_det.pt --device mps
# --reuse-tracks  пропустить детекцию, если tracks.json уже есть
```

Отчет → `reports/audit_<видео>.{json,md}`: итоги, разбивка по брендам, группам,
моделям и раздел «спорные зоны» (нужна проверка человеком) с кадрами-кропами.

Прогнать на нескольких видео сразу: `scripts/run_final_audit.sh`.

## Ограничения и путь к высокой точности

- Детектор — baseline на неидеальных авто-метках; полнота на плотных полках
  ограничена. Улучшается ручной проверкой части разметки и дообучением.
- `reference_images` в каталоге пусты → визуального поиска SKU пока нет, работает
  только OCR. Съемка 3–5 эталонных фото на модель (для товаров с нечитаемым
  текстом) — главный рычаг к «почти 100%» вместе с разделом ручной проверки.
