# Tablet live feedback

Планшет не должен пытаться делать полный SKU recognition на каждом кадре. Его
задача в live-режиме: помочь менеджеру снять каждый физический товар достаточно
четко, а точный бренд/модель оставить backend-пайплайну.

## Текущее изменение

Добавлен `LiveCaptureTracker`:

- связывает соседние detections в короткие tracks по IoU;
- оценивает каждый бокс по confidence TFLite-модели и локальной резкости кропа;
- если один физический объект несколько кадров подряд снят хорошо, track
  считается `captured`;
- captured-товар больше не рисуется рамкой, чтобы менеджер видел только то, что
  осталось доснять;
- поверх камеры показываются счетчики `Осталось`, `Готово`, `Проблемы`;
- кнопка `Сброс снятого` очищает live-память перед новой полкой/проходом.

Это привязка к физическому объекту внутри live-сессии, не к SKU-коду. Привязка к
распознанному SKU появится после backend-распознавания или после добавления
легкого on-device retrieval/OCR.

## Стабильная привязка к товару (re-id)

Раньше привязка была только по экранному IoU: когда камера уходила и
возвращалась, уже снятый товар появлялся в другом месте кадра, не совпадал по
IoU и снова обводился рамкой. Теперь у каждого товара есть **визуальная подпись**
— компактный отпечаток кропа (mean-removed, L2-normalized 8×8 luma, 64 числа,
`MainActivity.signatureFromLuma`). `LiveCaptureTracker` сопоставляет детекции с
треками жадно один-к-одному по двум сигналам:

- **позиция** (IoU) — непрерывность кадр-в-кадр;
- **внешний вид** (косинус подписи) — повторное опознание после движения камеры.

Поэтому уже снятый товар, появившись в другом месте кадра, опознаётся и **не
обводится повторно**. Снятые товары держатся в памяти дольше
(`KEEP_CAPTURED_MISSED`), а вместо рамки им рисуется маленькая зелёная ✓
(`ProductOverlayView.setCapturedMarks`), чтобы менеджер видел: «этот готов».
Подпись замораживается в момент `captured` — для стабильного re-id. Это всё ещё
привязка к физическому товару on-device (без чтения SKU), но устойчивая к
движению камеры.

## Обновление обратной связи

Live-рамка теперь говорит менеджеру, что именно исправить:

- зелёный `держите` — товар крупный, резкий и уверенный; если держать несколько
  кадров, он попадёт в `Готово` и рамка исчезнет;
- оранжевый `ближе` — товар найден, но слишком мелкий для чтения артикула;
- красный `медленнее` — crop смазан, нужно вести планшет медленнее;
- синий `наведите` — товар неуверенный или не в фокусе, нужно центрировать.

Внизу камеры показывается главный следующий шаг, например
`Подойдите ближе: товар мелкий для артикула`, плюс краткая разбивка проблем:
`ближе N, резче M, навести K`. Верхние счетчики стали короче:
`Осталось`, `Готово`, `Проблемы`.

Качество считается локально без OCR:

- резкость crop-а по Laplacian score;
- размер bbox в кадре (`area` и минимальная сторона), чтобы не принимать слишком
  дальний товар как хороший;
- confidence TFLite-детектора.

## Следующий UX-слой

Нужен экран, где основной первый взгляд менеджера:

- `Осталось: N`;
- `Готово: M`;
- отдельные причины проблем: `ближе`, `медленнее`, `наведите`;
- зеленые рамки показываются коротко и исчезают;
- мини-карта покрытия остается отдельным режимом или компактной панелью.

Лучший интерфейс для менеджера: меньше текста, больше состояний и счетчиков.
Live-камера должна говорить: "эти товары уже ок, вот эти пересними".
